# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] Ejecuté el programa al menos una vez sin errores críticos.
- [ ] Probé una entrada válida y confirmé la salida esperada.
- [ ] Probé una entrada inválida y documenté la respuesta.
- [ ] Verifiqué un caso límite simple.

## 2) Checklist de documentación
- [ ] Completé `docs/propuesta.md`.
- [ ] Completé `docs/caso_de_uso.md`.
- [ ] Revisé `docs/estructura_repositorio.md` y mantuve orden.
- [ ] Completé `docs/plan_de_pruebas.md`.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` detecta correctamente mi archivo principal.
- [ ] Mi comando principal de ejecución quedó documentado.
- [ ] No usé dependencias complejas ni servicios externos.

## 4) Checklist de entrega final
- [ ] El proyecto se mantiene pequeño y viable.
- [ ] La documentación explica claramente problema, alcance y solución.
- [ ] Incluí evidencia de pruebas realizadas.
- [ ] Realicé commit final con todos los archivos requeridos.
